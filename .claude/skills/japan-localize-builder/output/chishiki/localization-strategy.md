# チシキ (Chishiki) — ローカライズ戦略ドキュメント

## 1. 元プロダクト完全分析（Rabrain）

### 全機能リスト
| 機能 | UI構成 | 詳細 |
|------|--------|------|
| AIブックマーク保存 | Chrome拡張＋Webアプリ | ワンクリックでWebページ・記事・PDFを保存 |
| AI要約生成 | 記事詳細画面の上部 | ChatGPT生成の記事サマリーを自動表示 |
| セマンティック検索 | 検索バー（グローバルヘッダー） | 意味ベースで保存コンテンツを横断検索 |
| PDF閲覧＋注釈 | PDF専用ビューア | PDF直接閲覧、インラインノート追加 |
| ボイスメモ | 録音ボタン＋テキスト変換 | 音声→テキスト変換、メモとして保存 |
| タグ＋フォルダ管理 | サイドバー | 自動タグ＋手動フォルダ分類 |
| RAG対話 | チャットUI | 保存コンテンツについてAIに質問 |
| Pocket/Omnivore移行 | 設定画面 | 既存データインポート |
| ブラウザ拡張 | Chrome拡張 | ブラウザからワンクリック保存 |

### ビジネスモデル
- フリーミアム（無料枠＋有料プレミアム）
- 月額サブスクリプション

### ユーザージャーニー
1. **登録** → メール/Google認証でサインアップ
2. **オンボーディング** → Chrome拡張インストール案内、初回ブックマーク保存体験
3. **コア体験** → ブックマーク保存→AI要約→検索→RAG対話
4. **課金** → 無料枠を超えた保存数/AI利用回数で有料プラン案内
5. **リテンション** → 保存量が増えるほどセマンティック検索の価値が向上

### 成功要因
- Pocket/Omnivoreの終了・停滞を受けた移行需要
- AI要約とセマンティック検索という明確な差別化
- "第二の脳"というコンセプトの訴求力

## 2. 日本市場ギャップ分析

### 日本の類似サービスと弱点
| サービス | 弱点 |
|----------|------|
| Raindrop.io | AI機能なし、日本語最適化不十分 |
| Pocket | 開発停滞、AI機能なし、UIが古い |
| Notion | 汎用すぎ、ブックマーク管理が本業でない |
| Obsidian | テック層限定、ブックマーク管理ではない |
| はてなブックマーク | ソーシャルブックマーク（個人ナレッジ管理ではない） |

### 日本ユーザーの未充足ニーズ
- 日本語記事のAI要約（日本語の自然な要約）
- Qiita、Zenn、Note等の日本語テック記事の最適保存
- 「あとで読む」の整理が追いつかない問題
- 保存した情報を活用できていない問題

### 文化的に変更が必要な点
- 丁寧な日本語UI（敬語ベースの説明文）
- より細かいカテゴリ分類（日本人は分類好き）
- 学習・仕事・趣味といった「用途別」整理
- おすすめ機能（日本人は「おすすめ」に弱い）

## 3. ローカライズ方針

### UI/UX変更点
- **レイアウト**: サイドバー＋メインコンテンツの2カラム（Rabrainと同様）
- **情報密度**: やや高め（日本人は情報量を好む）
- **色彩**: インディゴ/ネイビー系メインカラー（知性・信頼感）
- **フォント**: Noto Sans JP（メイン）、BIZ UDPGothic（代替）
- **角丸**: 適度な角丸（8px-12px）で親しみやすさ

### コピーライティング方針
- **トーン**: 丁寧だがカジュアル（です・ます調）
- **敬語レベル**: B2C向け、フレンドリーな敬語
- **キャッチコピー案**:
  1. 「あなたの知識を、AIが整理する」
  2. 「読んだ、忘れた、をなくそう。」
  3. 「保存した情報が、知恵になる。」
  4. 「ブックマークに、AIの脳を。」

### 決済手段優先順位
1. クレジットカード（Stripe）
2. コンビニ払い（将来対応）
3. PayPay（将来対応）

### 日本独自機能
- 日本語セマンティック検索の最適化
- Qiita/Zenn/Note記事の最適化パーサー
- 日本語AI要約の品質チューニング
- 「学習」「仕事」「趣味」用途別コレクション

### 法的対応
- 特定商取引法に基づく表記
- プライバシーポリシー（個人情報保護法準拠）
- 利用規約
- インボイス制度対応（将来的）
